# Impact Map: Pluggable Prediction Engine — MCOA

> Generated by Agentic SDLC Phase 1 — verified against actual codebase on feature/rs-prediction-engine branch

## Summary

Everything under `internal/analytics/rightsizing/prediction/` is **greenfield** — no prediction code exists today.
Integration touches ~15 existing files across addon options, handlers, scrapeconfig, Helm templates, COO/Perses, and main.go.

## 1. New Files (~35 files across 8 sub-packages)

### prediction/ root (6 files)
- `prediction/types.go` — shared domain types (ForecastRequest/Result, DataPoint, config structs)
- `prediction/holt_winters.go` — triple exponential smoothing
- `prediction/stl.go` — STL decomposition via LOESS
- `prediction/autoregressive.go` — AR(p) via Yule-Walker
- `prediction/backtest.go` — train/validation split, MAPE/RMSE
- `prediction/ensemble.go` — weighted combiner, inverse-MAPE weights

### prediction/provider/ (6 files)
- `provider/interface.go` — PredictionProvider contract + enums
- `provider/registry.go` — factory from ADC config to concrete provider
- `provider/builtin/provider.go` — wraps ensemble, no network
- `provider/onnx/provider.go` — onnxruntime-go inference
- `provider/external/provider.go` — HTTP to vendor APIs + consent + redaction
- `provider/custom/provider.go` — REST/gRPC to customer server

### prediction/features/ (6 files)
- `features/types.go` — FeatureVector, FeatureConfig, ExtractFeatures
- `features/temporal.go` — time/calendar features
- `features/statistical.go` — rolling stats, percentiles
- `features/trend.go` — slope, acceleration, changepoint
- `features/workload.go` — burst/idle/efficiency
- `features/correlation.go` — cross-series correlations

### prediction/anomaly/ (5 files)
- `anomaly/types.go` — AnomalyResult, detector config
- `anomaly/zscore.go` — residual z-score
- `anomaly/rateofchange.go` — derivative thresholds
- `anomaly/adaptive.go` — adaptive percentile thresholds

### prediction/optimizer/ (3 files)
- `optimizer/types.go` — OptimizationResult, savings, bounds

- `optimizer/savings.go` — savings estimation helpers

### prediction/training/ (4 files)
- `training/types.go` — TrainingConfig, WorkloadKey, shard metadata
- `training/querier.go` — Thanos range-query adapter
- `training/validator.go` — backtest validation gate

### prediction/privacy/ (4 files)
- `privacy/consent.go` — consent gate + label redaction
- `privacy/audit.go` — audit events/metrics
- `privacy/rbac.go` — RBAC object helpers

### Helm templates (3 new files)

## Decisions (Human Checkpoint 1 — Approved)

1. **API structure**: Prediction is a flat sibling field under MCO's `PlatformAnalyticsSpec` (no breaking YAML change).
2. **Metric naming**: Use `acm_rs:prediction_*` (recording rule style with colons) consistently across all code.
3. **Delegation**: Prediction ADC sync is independent of `delegatingToMCOA` — always runs in MCOA regardless of RS mode.
